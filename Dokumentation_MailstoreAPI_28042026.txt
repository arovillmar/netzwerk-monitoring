PHASE 4: ERSTER API-TEST (GetServerInfo)
------------------------------------------------------------------------

BEFEHL 7 (fehlgeschlagen – Zertifikatsfehler):
  Invoke-RestMethod -Uri "https://192.168.80.120:8463/api/invoke/GetServerInfo" `
    -Method POST -Credential $cred

  Beschreibung: HTTP POST an die MailStore API. Ohne -SkipCertificateCheck
  schlägt es fehl, weil MailStore ein selbstsigniertes Zertifikat verwendet.
  Ergebnis: "RemoteCertificateNameMismatch, RemoteCertificateChainErrors"

BEFEHL 8 (Splatting-Methode – empfohlen für lange Befehle):
  $params = @{
    Uri                  = "https://192.168.80.120:8463/api/invoke/GetServerInfo"
    Method               = "POST"
    Credential           = $cred
    SkipCertificateCheck = $true
  }
  Invoke-RestMethod @params

  Beschreibung: Hashtable-Splatting übergibt alle Parameter als @-Variable.
  Vermeidet Zeilenumbruch-Probleme bei langen Befehlen.
  Parameter:
    SkipCertificateCheck  Ignoriert selbstsignierte/ungültige Zertifikate
  Ergebnis: Erfolg – statusCode: "succeeded", version: "26.1.0.23845",
            machineName: "SASCHA_SERVER"

BEFEHL 9 (BOM-Problem gelöst):
  (Invoke-RestMethod @params).TrimStart([char]0xFEFF) | ConvertFrom-Json

  Beschreibung: MailStore sendet ein UTF-8 BOM (Byte Order Mark, Zeichen 0xFEFF)
  am Anfang jeder JSON-Antwort. Dies verhindert die automatische JSON-Konvertierung.
  TrimStart entfernt das BOM, danach kann ConvertFrom-Json korrekt parsen.

------------------------------------------------------------------------
ALLGEMEINE BEGRIFFE
------------------------------------------------------------------------

Administration API
  Schnittstelle zum Fernsteuern eines Servers über HTTP-Anfragen. Bei MailStore
  wird die Administration API verwendet, um Serverinformationen abzurufen,
  Archivierungsjobs zu steuern und Ergebnisse auszuwerten – ohne grafische
  Oberfläche oder installiertes Zusatzmodul.

API (Application Programming Interface)
  Definierte Schnittstelle, über die Programme miteinander kommunizieren.
  Die MailStore Administration API verwendet HTTP als Transportprotokoll
  und JSON als Datenformat.

Basic Authentication (HTTP Basic Auth)
  Einfaches Authentifizierungsverfahren: Benutzername und Passwort werden
  als Base64-kodierter String im HTTP-Header "Authorization" übertragen.
  Format: "Basic " + Base64(Benutzername:Passwort)
  Hinweis: Nur sicher in Kombination mit HTTPS (verschlüsselte Verbindung).

BOM (Byte Order Mark)
  Unsichtbares Sonderzeichen (Unicode U+FEFF) am Anfang einer UTF-8-Datei
  oder -Antwort. Gibt dem Leseprogramm das Zeichenkodierungsformat an.
  MailStore schickt ein BOM in jeder API-Antwort, was PowerShells
  automatische JSON-Konvertierung stört. Lösung: .TrimStart([char]0xFEFF)

ConvertFrom-Json
  PowerShell-Cmdlet, das einen JSON-String in ein PowerShell-Objekt umwandelt.
  Scheitert wenn der String mit einem BOM-Zeichen beginnt.

ConvertTo-Json
  PowerShell-Cmdlet, das ein PowerShell-Objekt in JSON-Format umwandelt.
  Parameter -Depth legt fest wie tief verschachtelte Objekte dargestellt werden.

HTTP POST
  Eine HTTP-Anfragemethode bei der Daten an den Server gesendet werden.
  Die MailStore API verwendet ausschließlich POST für alle Aufrufe.
  Gegensatz zu GET: POST kann Daten im Anfrage-Body mitschicken.

HTTPS
  HTTP Secure – verschlüsselte Version des HTTP-Protokolls.
  Verwendet TLS/SSL-Zertifikate zur Verschlüsselung der Verbindung.
  MailStore verwendet HTTPS auf Port 8463 mit einem selbstsignierten Zertifikat.

Invoke-RestMethod
  PowerShell-Cmdlet für HTTP-Anfragen. Konvertiert JSON-Antworten automatisch
  zu PowerShell-Objekten (wenn kein BOM stört). Unterstützt alle HTTP-Methoden.

JSON (JavaScript Object Notation)
  Textbasiertes Dateiformat für strukturierte Daten. Einfach lesbar für
  Menschen und Maschinen. MailStore API-Antworten kommen immer als JSON.
  Beispiel: {"statusCode": "succeeded", "result": {"version": "26.1.0.23845"}}

MS.PS.Lib.psd1
  PowerShell-Modulbeschreibungsdatei der MailStore Server PowerShell-Bibliothek.
  Teil des "MailStore Server Scripting Tutorial" – einem separaten Download von
  deepinvent. Nicht im Standard-Installationsumfang von MailStore Server enthalten.
  Alternative: Direkte HTTP API-Aufrufe ohne dieses Modul.

PowerShell Remoting (WinRM)
  Windows-Dienst der es ermöglicht, PowerShell-Befehle auf einem entfernten
  Computer auszuführen (New-PSSession, Invoke-Command). Muss explizit aktiviert
  und konfiguriert werden. Auf der Mailstore-VM 192.168.80.120 nicht erreichbar.

SecureString
  PowerShell-Datentyp für verschlüsselte Strings (z.B. Passwörter).
  Wird von Get-Credential zurückgegeben. Schützt Passwörter im Arbeitsspeicher.
  Für API-Aufrufe muss der SecureString in einen Klartext-String umgewandelt
  werden: [Marshal]::SecureStringToBSTR() + [Marshal]::PtrToStringAuto()

Self-Signed Certificate (Selbstsigniertes Zertifikat)
  Ein TLS-Zertifikat das nicht von einer offiziellen Zertifizierungsstelle
  ausgestellt wurde. MailStore erstellt bei der Installation automatisch ein
  solches Zertifikat. Browser und PowerShell vertrauen ihm standardmäßig nicht.
  Lösung: -SkipCertificateCheck in Invoke-RestMethod

Splatting
  PowerShell-Technik bei der Parameter als Hashtable (@{}) gesammelt und dann
  mit @variablenname an ein Cmdlet übergeben werden. Vermeidet sehr lange
  Befehlszeilen die beim Kopieren umbrechen.

TCP (Transmission Control Protocol)
  Verbindungsorientiertes Netzwerkprotokoll. Stellt sicher dass Datenpakete
  vollständig und in richtiger Reihenfolge ankommen. HTTP/HTTPS laufen über TCP.

Test-NetConnection
  PowerShell-Cmdlet das eine Netzwerkverbindung zu einem Host und Port testet.
  Gibt TcpTestSucceeded zurück (True/False). Ähnlich wie "telnet" oder "ping"
  kombiniert, aber mit Port-Prüfung.

TLS (Transport Layer Security)
  Verschlüsselungsprotokoll für sichere Netzwerkverbindungen. Nachfolger von SSL.
  HTTPS verwendet TLS um die Verbindung zu verschlüsseln.

URI-Kodierung (URL-Encoding)
  Sonderzeichen in URLs müssen kodiert werden damit sie korrekt übertragen werden.
  Beispiel: Leerzeichen → %20, / → %2F, : → %3A
  PowerShell: [Uri]::EscapeDataString($string)
  Wichtig: + muss als %2B kodiert werden, sonst wird es als Leerzeichen interpretiert.

UTF-8
  Zeichenkodierungsstandard der alle Unicode-Zeichen darstellen kann.
  Standard für Web-APIs und JSON. MailStore verwendet UTF-8 mit BOM.

------------------------------------------------------------------------
MAILSTORE-SPEZIFISCHE BEGRIFFE
------------------------------------------------------------------------

MailStore Administration API
  HTTP REST-Schnittstelle von MailStore Server für administrative Aufgaben.
  Standardmäßig deaktiviert – muss in der "MailStore Server Service Configuration"
  aktiviert werden. URL-Schema: https://<server>:8463/api/invoke/<Funktion>

MailStore admin-Benutzer
  Der in MailStore integrierte Administrator. NICHT identisch mit dem
  Windows-Benutzer "Administrator". Wird für API-Aufrufe benötigt.
  Das Passwort wird in MailStore selbst verwaltet (nicht Windows-Passwort).

MailStore Server Service Configuration
  Separates Windows-Programm (nicht der Management Client!) zur Konfiguration
  des MailStore-Dienstes. Hier wird die Administration API aktiviert.
  Zu finden im Windows-Startmenü unter "MailStore Server Service Configuration"
  oder im Installationsverzeichnis als MailStoreServerServiceConfiguration.exe

MailStore Management Client
  Die normale Benutzeroberfläche für MailStore-Administratoren. Hier können
  Archivierungsprofile, Jobs und Speicherorte verwaltet werden.
  Davon getrennt: Die Administration API für automatisierte Abfragen.

GetServerInfo
  API-Funktion: Gibt Basisinformationen zum MailStore-Server zurück.
  Keine Parameter erforderlich. Antwortfelder: version, machineName
  Verwendung zum Testen ob API erreichbar und Credentials korrekt sind.

GetJobResults
  API-Funktion: Gibt Ausführungsergebnisse von Archivierungsjobs zurück.
  Pflichtparameter:
    fromIncluding  Startzeit (Format: yyyy-MM-ddTHH:mm:ss)
    toExcluding    Endzeit   (Format: yyyy-MM-ddTHH:mm:ss)
    timeZoneId     Zeitzone  (z.B. "W. Europe Standard Time")
  Optional: jobId (nur Ergebnisse eines bestimmten Jobs)
  Antwortfelder: jobResultId, jobId, action, startTime, completeTime, result

GetJobs
  API-Funktion: Gibt alle konfigurierten Archivierungsjobs zurück.
  Keine Parameter. Antwortfelder: id, name, action, owner, schedule
  Nützlich um Job-IDs mit sprechenden Namen zu verknüpfen.

GetTimeZones
  API-Funktion: Gibt alle gültigen Timezone-IDs zurück die der Server kennt.
  Keine Parameter. Antwortfelder: id, displayName
  Deutschland: id = "W. Europe Standard Time"

get-metadata
  API-Endpunkt (nicht invoke/): Gibt Metadaten aller API-Funktionen zurück.
  URL: https://<server>:8463/api/get-metadata
  Unverzichtbar um Pflichtparameter, Datentypen und Funktionsnamen zu ermitteln.

RunProfile
  MailStore-Aktion die ein Archivierungsprofil ausführt.
  Format in API: "RunProfile --id=<ProfilNummer>"
  Archivierungsprofile legen fest welche E-Mail-Konten archiviert werden.

VerifyStores
  MailStore-Aktion die die Integrität aller Archivspeicher prüft.
  Läuft typischerweise nachts und kann Stunden dauern.
  Bei Fehler: MailStore Management Client → Verwaltung → Speicherorte → Protokoll

Archivierungsjob (Job)
  Eine geplante Aufgabe in MailStore die automatisch zu festgelegten Zeiten
  ausgeführt wird. Beispiele: E-Mails archivieren, Backup erstellen,
  Integritätsprüfung durchführen, Statusbericht senden.

Job-Status (result-Feld)
  succeeded  Job erfolgreich abgeschlossen
  failed     Job mit Fehler abgebrochen – Untersuchung erforderlich
  cancelled  Job wurde abgebrochen (z.B. durch gleichzeitigen anderen Job)

------------------------------------------------------------------------
POWERSHELL-BEFEHLE UND PARAMETER
------------------------------------------------------------------------

Get-Credential
  Öffnet einen sicheren Passwort-Eingabedialog.
  Gibt ein PSCredential-Objekt zurück (Benutzername + SecureString-Passwort).
  Parameter: -UserName (Vorausfüllung), -Message (Anzeigetext)

Get-ChildItem
  Listet Dateien und Ordner auf. Mit -Recurse durchsucht es Unterordner.
  Parameter: -Filter (Dateinamenfilter), -ErrorAction SilentlyContinue (Fehler ignorieren)

Import-Module
  Lädt ein PowerShell-Modul (.psd1 oder .psm1 Datei) in die Session.
  Fehler wenn Datei nicht existiert oder ungültig ist.

New-PSSession
  Erstellt eine persistente PowerShell-Remoting-Verbindung zu einem Remote-Computer.
  Erfordert WinRM auf dem Zielcomputer und gültige Credentials.

Select-Object -ExpandProperty
  Gibt den Inhalt einer einzelnen Eigenschaft zurück (nicht das Wrapper-Objekt).
  Beispiel: | Select-Object -ExpandProperty result  → gibt result-Array direkt zurück

SkipCertificateCheck
  Parameter von Invoke-RestMethod (PowerShell 7+): Ignoriert Zertifikatsfehler.
  Notwendig bei selbstsignierten Zertifikaten wie bei MailStore.
  Nur für interne/vertrauenswürdige Server verwenden.

Test-Connection
  PowerShell-Equivalent zu "ping". Prüft ob ein Host im Netzwerk erreichbar ist.
  Parameter: -Count (Anzahl Pakete), -TimeoutSeconds (Wartezeit)

Test-Path
  Prüft ob eine Datei, ein Ordner oder ein Registry-Pfad existiert.
  Gibt $true oder $false zurück.

TrimStart
  .NET-Methode die bestimmte Zeichen vom Anfang eines Strings entfernt.
  Verwendung: $string.TrimStart([char]0xFEFF) entfernt das UTF-8 BOM-Zeichen.

Where-Object
  Filtert eine Liste von Objekten anhand einer Bedingung.
  Kurzform: ? { $_.Eigenschaft -eq "Wert" }

------------------------------------------------------------------------
NETZWERK UND PROTOKOLLE
------------------------------------------------------------------------

Port 8463
  Standard-HTTPS-Port der MailStore Administration API.
  Hinweis: In älteren Dokumentationen findet sich manchmal Port 8474 – das ist falsch
  bzw. veraltet. Der korrekte Port für MailStore Server ist 8463.

Port 3389
  Standard-RDP-Port (Remote Desktop Protocol).
  Wird für Windows-Fernzugriff verwendet.
  Im Monitor geprüft um festzustellen ob die Windows-VM erreichbar ist.

Port 9522
  Speedwire-Port des SMA Home Manager (Photovoltaik-Monitoring).
  Verwendet UDP, nicht TCP! Ein TCP-Test auf diesem Port ist daher immer False.

SSH (Secure Shell)
  Verschlüsseltes Protokoll für Fernzugriff auf Linux/Unix-Systeme (Port 22).
  Wird im Monitor verwendet für: Raspberry Pi, Synology NAS (Port 822).
  Ermöglicht Ausführung von Befehlen auf Remotesystemen.

SMTP (Simple Mail Transfer Protocol)
  Protokoll zum Versenden von E-Mails. Standard-Port: 587 (mit TLS/STARTTLS).
  Wird vom Monitor für das Versenden von Alarmierungen und Berichten verwendet.

WinRM (Windows Remote Management)
  Windows-Dienst der PowerShell Remoting ermöglicht.
  Muss explizit aktiviert werden (Enable-PSRemoting).
  Auf der MailStore-VM 192.168.80.120 nicht verfügbar (Zugriff verweigert).

credentials.json

Splatting (@params)
  Technik um lange PowerShell-Befehle übersichtlich zu halten.
  Parameter werden in einer Hashtable gesammelt und mit @variablenname übergeben.
  Verhindert Zeilenumbruch-Probleme beim Kopieren aus der Dokumentation.

Setup-Credentials.ps1
  Interaktives Skript zum sicheren Eingeben und Speichern aller Passwörter
  für den Heimnetz-Monitor. Erstellt credentials.json.

Start-NetworkMonitor.ps1

Send-Sofortbericht.ps1

  Nützlich für manuelle Checks zwischen den automatischen Intervallen.


================================================================================
  ANHANG – BEKANNTE PROBLEME UND LÖSUNGEN
================================================================================

Problem: MailStore API gibt "String was not recognized as a valid DateTime"
Lösung:  Datumsformat muss exakt "yyyy-MM-ddTHH:mm:ss" sein (T zwischen Datum/Zeit)

Problem: "Missing API argument fromIncluding"
Lösung:  GetJobResults benötigt ALLE drei Parameter: fromIncluding, toExcluding, timeZoneId

Problem: ConvertFrom-Json schlägt fehl mit BOM-Zeichen
Lösung:  (Invoke-RestMethod @p).TrimStart([char]0xFEFF) | ConvertFrom-Json

Problem: "401 Unauthorized"
Lösung:  Falsches Passwort ODER falsche Credentials-Variable ($cred veraltet)
         Neu eingeben: $cred = Get-Credential -UserName "admin"

Problem: "Connection refused" auf Port 8463
Lösung:  API in MailStore Server Service Configuration aktivieren (Häkchen setzen)
         Danach: MailStore-Dienst neu starten

Problem: Befehl wird beim Einfügen umgebrochen und schlägt fehl
Lösung:  Befehle in kurze einzelne Zeilen aufteilen, Splatting (@{}) verwenden
         Lange Strings vorher in Variablen speichern

Problem: "RemoteCertificateNameMismatch"
